# 离线链路（Ingestion Pipeline）缺失项清单

## 一、核心流程缺失（高优先级）

| 缺失项 | 说明 | 涉及文件 |
|--------|------|----------|
| 文档更新流程 | `DocumentLifecycleService.updateDocument()` 抛 `UnsupportedOperationException`，"先写新再删旧"策略未实现。Controller 也没有更新接口。 | `DocumentLifecycleService.java` |
| 任务查询/重试/取消 | `TaskService` 中 `queryTask`、`queryTasks`、`retry`、`cancel` 全是 TODO。 | `TaskService.java` |
| 后台任务调度 | 设计文档要求"从 task 表中按 PENDING 状态拉取任务"，当前是 Controller 直接触发，没有后台轮询调度器。 | — |
| 断点续传 | 设计文档要求"从上次失败的 stage 开始，跳过已完成的阶段"。当前 `retry` 未实现，`IngestionPipeline` 也不支持从指定 stage 恢复。 | `IngestionPipeline.java` |

---

## 二、可靠性机制缺失（中优先级）

| 缺失项 | 说明 |
|--------|------|
| 索引补偿机制 | 双写失败时当前仅标记 `INDEX_PARTIAL`，缺少后台补偿队列/定时任务去修复。 |
| 定时对账任务 | 设计文档要求"比对元数据库与各索引存储的数据量，发现不一致自动修复"。 |
| 超时控制 | 设计文档要求"单文档处理超时 5 分钟"。 |
| 重试策略 | 设计文档要求"指数退避：30s → 60s → 120s"，当前流水线内无重试。 |
| 死信队列 | 设计文档要求"解析失败进入死信队列，支持人工介入重试"。 |
| 并发锁 | 设计文档要求"乐观锁或分布式锁"防止同一任务被多线程重复处理。 |

---

## 三、存储实现不完整

| 缺失项 | 说明 | 涉及文件 |
|--------|------|----------|
| 内存向量检索 stub | `InMemoryVectorStore.search()` 返回空列表，未实现余弦相似度计算。 | `InMemoryVectorStore.java` |
| 内存 BM25 检索 stub | `InMemoryBM25Store.search()` 返回空列表，未实现 BM25 计算。 | `InMemoryBM25Store.java` |

---

## 四、扩展能力缺失（低优先级 / 后续演进）

| 缺失项 | 说明 |
|--------|------|
| 多数据源接入 | 当前仅支持本地文件上传。设计中的网页/Wiki、数据库、API、日志等数据源均未实现。 |
| 监控告警 | 任务失败率、索引写入延迟、补偿队列堆积等告警机制未实现。 |

---

## 总结

当前离线链路的**主干流程已跑通**（文件上传 → 去重 → 解析 → 结构化 → Chunk切分 → Embedding → 双写索引），但围绕**任务治理**（查询/重试/取消/调度）、**可靠性保障**（补偿/对账/超时/重试/死信）、以及**检索 stub 补全**这三块还有明显缺口。如果目标是 MVP 跑通，可以优先补齐文档更新流程和任务查询 API；如果要上生产，则需要把可靠性机制一并补上。
